## 6.1 如何获取帮助

### 6.1.1 自助排查

遇到问题，建议先按以下顺序排查：

【信息框 - 蓝色】
📌 **排查顺序**

1. 📖 **查文档** - 先看本文档相关章节
2. 🔍 **查日志** - `openclaw gateway logs`
3. 🩺 **运行诊断** - `openclaw doctor`
4. 🐛 **搜索 Issues** - GitHub 上搜索类似问题

### 6.1.2 提问渠道

【表格】
| 渠道 | 适用场景 | 响应时间 |
|------|----------|----------|
| **GitHub Issues** | Bug 报告、功能建议 | 24-48h |
| **飞书群** | 快速提问、交流讨论 | 实时 |
| **邮件** | 私密问题、商务合作 | 48h |
| **文档评论** | 文档相关疑问 | 24h |

### 6.1.3 提交 GitHub Issue

【折叠块：Issue 模板】

**Bug 报告：**
```markdown
## 问题描述
清晰描述遇到的问题

## 环境信息
- OS: macOS 14.1 / Ubuntu 22.04
- Python: 3.11.0
- OpenClaw: 2026.2.2
- 工具版本: 1.0.0

## 复现步骤
1. 运行 '...'
2. 点击 '...'
3. 出现错误

## 期望结果
应该发生什么？

## 实际结果
发生了什么？附上错误日志

## 日志
```
粘贴相关日志
```
```

**功能建议：**
```markdown
## 功能描述
想实现什么功能

## 使用场景
在什么情况下需要

## 期望行为
具体希望怎样工作

## 可能的实现
如果有想法可以分享
```

## 6.2 贡献指南

### 6.2.1 如何贡献

欢迎各种形式的贡献！

【表格】
| 贡献类型 | 说明 | 难度 |
|----------|------|------|
| 🐛 **报告 Bug** | 发现问题并提交 Issue | ⭐ |
| 💡 **功能建议** | 提出新想法 | ⭐ |
| 📝 **改进文档** | 修复 typo、补充说明 | ⭐⭐ |
| 🔧 **提交代码** | 修复 Bug 或实现功能 | ⭐⭐⭐ |
| 🎨 **设计资源** | Logo、插图、UI 设计 | ⭐⭐ |
| 🌐 **翻译** | 将文档翻译成其他语言 | ⭐⭐ |

### 6.2.2 代码贡献流程

【折叠块：PR 流程】
1. **Fork 仓库**
   - 点击 GitHub 上的 Fork 按钮

2. **克隆代码**
   ```bash
   git clone https://github.com/YOUR_NAME/openclaw-feishu-deployer.git
   cd openclaw-feishu-deployer
   ```

3. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **修改代码**
   - 编写代码
   - 添加测试
   - 更新文档

5. **提交更改**
   ```bash
   git add .
   git commit -m "🦞 feat: 添加 xxx 功能"
   git push origin feature/your-feature-name
   ```

6. **创建 PR**
   - 在 GitHub 上点击 "New Pull Request"
   - 填写 PR 描述
   - 等待 Review

### 6.2.3 代码规范

【信息框 - 黄色】
⚠️ **提交信息格式：**
```
🦞 type: 简短描述

type 类型：
- 🦞 feat: 新功能
- 🐛 fix: Bug 修复
- 📚 docs: 文档更新
- 💅 style: 代码格式
- ♻️ refactor: 重构
- 🧪 test: 测试
- 🔧 chore: 构建/工具
```
