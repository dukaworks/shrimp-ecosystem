### Phase 2: 工具扩展 🚧 (1-2个月)

**时间**: 2025-03 ~ 2025-04
**目标**: 完善工具链

#### 2.1 健康虾 🏥 (openclaw-health)

```yaml
命令: och / openclaw-health
功能:
  - 诊断 OpenClaw 运行状态
  - 检查配置文件
  - 测试 API 连接
  - 性能监控
  - 自动修复建议

使用场景:
  - "OpenClaw 不工作了？"
  - "检查一下哪里出了问题"
  - "优化性能"

命令示例:
  och check          # 全面检查
  och doctor         # 诊断问题
  och fix            # 自动修复
  och status         # 查看状态
  och logs           # 分析日志
```

#### 2.2 升级虾 ⬆️ (openclaw-upgrader)

```yaml
命令: ocg / openclaw-upgrade
功能:
  - 检查新版本
  - 一键升级
  - 升级前自动快照
  - 升级失败回滚
  - 版本管理

使用场景:
  - "升级到最新版"
  - "回滚到旧版本"
  - "查看更新日志"

命令示例:
  ocg check          # 检查更新
  ocg upgrade        # 执行升级
  ocg rollback       # 回滚版本
  ocg list           # 列出版本
```

#### 2.3 配置虾 ⚙️ (openclaw-config)

```yaml
命令: occ / openclaw-config
功能:
  - 交互式配置管理
  - 配置验证
  - 模板管理
  - 多配置切换
  - 配置导入/导出

使用场景:
  - "切换不同模型"
  - "管理多个配置"
  - "导出配置分享"

命令示例:
  occ edit           # 编辑配置
  occ validate       # 验证配置
 occ switch prod   # 切换到生产配置
  occ export        # 导出配置
```

### Phase 3: 六虾战队 🦞🦞🦞🦞🦞🦞 (3-6个月)

**时间**: 2025-05 ~ 2025-08
**目标**: 完整工具链

#### 工具矩阵

| 工具 | 代号 | 图标 | 职责 | 优先级 |
|------|------|------|------|--------|
| 部署虾 | ofd | 🦞 | 安装部署 | P0 ✅ |
| 卸载虾 | ocu | 💥 | 清理卸载 | P0 ✅ |
| 备份虾 | ocs | 💾 | 快照管理 | P0 ✅ |
| 健康虾 | och | 🏥 | 诊断监控 | P1 🚧 |
| 升级虾 | ocg | ⬆️ | 版本升级 | P1 🚧 |
| 配置虾 | occ | ⚙️ | 配置管理 | P1 🚧 |

#### 六虾协作流程

```
用户命令
    │
    ├───> ofd deploy ───> 部署
    │
    ├───> och check ────> 检查健康
    │
    ├───> ocs create ───> 创建快照
    │
    ├───> occ switch ───> 切换配置
    │
    ├───> ocg upgrade ──> 升级版本
    │     (自动调用 ocs 备份)
    │
    ├───> och doctor ───> 诊断修复
    │
    ├───> ocu ──────────> 卸载
    │     (自动调用 ocs 备份)
    │
    └───> ocs restore ──> 恢复
```

#### 统一 CLI 入口

```bash
# 主命令
shrimp --help           # 查看所有工具
shrimp deploy           # 调用 ofd
shrimp uninstall        # 调用 ocu
shrimp snapshot         # 调用 ocs
shrimp health           # 调用 och
shrimp upgrade          # 调用 ocg
shrimp config           # 调用 occ

# 快捷方式
scm deploy              # shrimp 的简写
```
